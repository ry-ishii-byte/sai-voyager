# フォルダ概要レポートを作成してoutputに出力

引数で指定されたフォルダ名（またはパス）の中身を調査し、概要レポートをMarkdownファイルとして `H:\共有ドライブ\SAIL\ai-management\output\` に保存してください。

## 手順

### Step 1: フォルダを特定する

引数 `$ARGUMENTS` をフォルダ名として受け取り、以下の順で場所を探してください。

1. `H:\共有ドライブ\SAIL\` 直下に完全一致するフォルダがあるか
2. 番号付きフォルダ名（例：`36_アントレベース`）として存在するか
3. フォルダ名の一部に引数が含まれるフォルダがあるか

見つからない場合は候補一覧をユーザーに提示して確認を求めてください。

### Step 2: フォルダ内のファイルを収集する

- フォルダ直下のサブフォルダ一覧と件数を把握する
- 読み取り可能なファイル（`.docx`, `.txt`, `.md`, `.csv`, `.xlsx`）を優先的に内容を読む
- PDFはテキスト抽出を試み、できなければファイル名から内容を推測する
- Googleドキュメント系（`.gdoc`, `.gsheet`, `.gslides`）はファイル名から内容を推測する

### Step 3: 概要レポートを作成する

以下の構成でMarkdownレポートを作成してください。

```
# [フォルダ名] 概要レポート

> 作成日：[今日の日付]
> 対象フォルダ：[フルパス]

## 概要
（このフォルダが何のためのものか、1〜3文で）

## フォルダ構成
（サブフォルダと主なファイルの一覧）

## 主なドキュメントの内容
（読み取れたファイルの要点をセクションごとに）

## わかったこと・まとめ
（事業・活動・目的などの総括）

## 備考
（読み取れなかったファイル、推測で補った部分などを明記）
```

### Step 4: outputフォルダに保存する

- ファイル名：`[フォルダ名]_概要レポート_[YYYYMMDD].md`
- 保存先：`H:\共有ドライブ\SAIL\ai-management\output\`
- 保存後、ファイルパスをユーザーに報告する
