## ファシリテーションのポイント

### アイスブレイク（「自分マップ」）

- 4マスを「得意・好き・過去の挑戦・未来の挑戦」に分けることで、現在（実力）と未来（欲求）が同時に見える。
- 「書けない」生徒への対応：
  - 得意なこと：「友達から『それうまいね』と言われたことは？」
  - 好きなこと：「休みの日に、何も予定がなければ何してる？」
  - 過去の挑戦：「最近ちょっと頑張ったこと、なんでもいい」
  - 未来の挑戦：「いつかやってみたいこと、現実的かどうか関係なく」
- チームシェア後に「共通点」と「違い」を発表させると、「違うことが当たり前」の多様性受容につながる。

### 導入講義（自己認識と自己効力感）

- 「自己認識」の日本語が難しい場合：「自分のトリセツ（取扱説明書）を持っている感覚」と説明すると子どもに響く。
- 「自己効力感」は「根拠のある自信」と説明する。「なんとなく自信がある」ではなく「過去の体験から来る確信」という違いを強調。
- スライド6の「4つの源泉」を板書に残しておき、ケーススタディでどの源泉が登場したかを振り返れるようにする。
- 「弱みを知ること」も自己認識の大事な要素であることを伝える（弱みは恥ずかしいものではなく、成長マップの一部）。

### ケーススタディの進め方

**ハルカの物語（自己認識→行動）**

- まずスライドを見せずに「得意なことが全くないという人はいる？」と問う。
- 手が挙がったら「実は、見えていないだけかもしれない。今から証拠を見せる」と導入する。
- ハルカの気づき（「人の話を聴く力」が強みだった）を提示した後：
  - 「あなたの場合、見えていない強みは何だと思う？」とグループで話し合わせる。
- まとめポイント：「自己認識は探すもの、磨くもの」

**リョウタの物語（小さな達成体験→自己効力感）**

- 「プレゼンや発表が怖いと感じたことがある人？」と問いかけて共感を引き出してから提示。
- 4段階のステップ表（スライド8）を板書で描きながら説明すると理解が深まる。
- 「リョウタが4回目にできた理由は何だと思う？」をグループで考えさせる。
  - 答え：「①小さな成功体験を積んだから」「②周囲の励ましを受けたから」= 源泉①と③
- 補足：「全校集会での2分スピーチは、最初から目標にしていなかった。小さな一歩の積み重ねがそこへ連れて行った」

### グループワークの進行

**フェーズ①（個人強みマップ）**

- 「弱み」の欄を最後に入れているのは意図的。最初に強みを書ききってから弱みに向き合うことで、弱みが「成長の余白」として見えやすくなる。
- 「ありがとうと言われた経験」欄は、自己認識の盲点（自分では気づかない貢献）を発見するための問い。書けない場合：「誰かに助けてあげたことは？」と言い換える。

**フェーズ②（チームの強み整理）**

- 「組み合わせると最強なこと」を引き出すのがポイント。
  - 例：「絵が得意な人」+「アイデアを言語化できる人」+「人を動かすのが好きな人」= プレゼン最強チーム
- 「このチームにしかできないこと」という問いへの答えが出にくい場合：
  - 「このメンバーを4人集めた理由を、神様目線で考えるとしたら？」と問う（ユニークな発想が出やすい）

**フェーズ③（アイデア開発）**

- 「強みを活かす」という制約を守らせる。「好きなアイデアを出す」ではなく「自分たちの強みが核になるアイデアを出す」が目的。
  - 強みとアイデアがつながっていないチームへ：「このアイデア、あなたの強みはどこで使われてる？」と問い返す。
- Step 2の「強みと問題をつなぐ」が最も詰まるポイント。巡回で「強みリストとアイデアの問題を声に出して読んでみて」と促す。

**フェーズ④（プロフィールシート完成）**

- 「チームのキャッチコピー」は楽しいポイントなので、最後まで残さず早めに決めるよう促す。これが決まるとシート全体が一気に完成する。
- 「チームの強みとの接続」欄は、プレゼン時に審査員が最も注目する欄として伝えると、丁寧に書いてくれる。

### プレゼンテーション

- 発表フォーマット（スライド15）を板書しておき、全グループが同じ構成で話せるようにする。
- 「強みの活かし方」を中心にフィードバックするよう聴衆に事前説明する：
  - 「アイデアの良し悪しより、強みがどう活かされているかを見てほしい」
- 付箋フィードバックは発表直後・沈黙で2分行う。「話しながら書かない」を徹底。
- 全体フィードバックでは：
  - 「最も多くのチームで使われた強みは何か」を集計して板書（例：「コミュニケーション力」が5チーム中4チームで出た = 全員が持つ共通資源）
  - 「最も独自な組み合わせを出したチーム」に注目してコメントする

### 振り返りシート

- 問1「今日気づいた強み」は、アイスブレイク・チームからのフィードバック・プレゼン後の付箋など、多くの場面から引き出せる。書けない場合：「付箋に書いてもらったことを思い出して」と誘導。
- 問3「明日から1つやってみること」は具体的かつ小さいものを推奨：
  - 「『得意なこと』を誰かに1文で伝える」
  - 「苦手だと思っていることを1回だけ試してみる」
  - 「チームメンバーの強みを1つ声に出して褒める」
- 問4「挑戦力との接続」：
  - 「自己効力感 = 『できる！』という気持ちが、挑戦への背中を押す。今日体験したことがまさにそれ」

## よくある問題と対処法

| 問題 | 対処 |
|------|------|
| 「得意なことがない」と本当に書けない | 「今日学校に来た——それが挑戦力！」「友達の話を最後まで聴く——それが傾聴力！」と日常を言語化する |
| チームの強みが「似たりよったり」になる | 「違う視点で見てみよう。○○さんが得意で△△さんが不得意なことは？」と比較させる |
| アイデアが強みと無関係になる | 「このアイデアに、あなたの強みをどう使う？」と問い、強みファーストで再考させる |
| プレゼンが全員で読み上げになる | 「担当を決めて、自分のパートだけ話してみよう」と役割分担を提案 |
| 振り返り問4（挑戦力との接続）が書けない | 「今日の30秒スピーチ（またはプレゼン）の直前と直後の気持ちを思い出して」と感情記憶を呼び起こす |
| 時間が足りなくてシートが未完成になる | フェーズ③まで完成していればプレゼンは口頭でOK。シートは参照資料として使う |
| チームの特定メンバーばかり発言する | 「今まで発言していない人から意見を聞こう」と促すか、ラウンドロビン形式（順番に発言）を導入 |

## SAILの5つの力との接続

| SAILの力 | このワークでの体験 |
|----------|------------------|
| **自律力** | 自分の強み・弱み・価値観を把握し、自分の意思でアイデアを形にする主体性を体験する |
| **挑戦力** | 「できない」という思い込みを外し、小さな達成体験を積み重ねて自己効力感を育てる |
| 創造力 | チームの強みを組み合わせて、個人では思いつかないアイデアを生み出す |
| 協働力 | メンバーの強みを積極的に引き出し、互いの可能性を広げるチームプロセスを体験する |
| 表現力 | 「自分はこういう人間だ」を言語化し、プレゼンとフィードバックで磨く |
