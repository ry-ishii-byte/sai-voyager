## ファシリテーションのポイント

### アイスブレイク

- 「不便ハンター」という言葉で「楽しそう」という雰囲気を作る。ゲーム感覚で始める。
- 「気づいたことがない」という参加者には「今朝起きてから今ここに来るまでに、何か面倒だったことは？」と時間軸を絞って問う。
- ファシリ自身が「不便だと感じて、解決策を考えたこと」を短く話すと、場が和らぐ。
- タイプ診断（スライド4）では「🔴が多くても正直でいい！今日でタイプが変わる」と伝える。

### 導入講義

**スライド6（問題 vs 機会）が最初の転換点。**
- 「問題を見たとき、イライラする人？」→「ワクワクする人？」と手を挙げさせる。
- 「今日ワクワク側の目線を体験してもらう」と目標を伝える。

**スライド7（価値の3種類）では自己アセスメントを促す。**
- 「自分が大切にしたい価値はどのタイプ？」と問い、理由も含めてチームでシェアさせる。
- 「どれが正解というわけではない。起業家によって重視する価値が違う」と補足する。

**スライド9（本当のニーズ）を実演する。**
- 「ドリルを買う人は穴が欲しい」の例を使い、実際に「なぜ？」を3回繰り返して見せる。
- 参加者から「学校のルールが多い → なぜ？ → 先生が管理しやすいから → なぜ？ → 〇〇」と連鎖させるデモをすると理解が深まる。

### ケーススタディ

- Aさんの「どうせ変わらない」という諦め感を丁寧に扱う。「Aさんが悪い人ではない。誰もがAさんになりうる。大事なのは"もう一歩考えてみる"こと」と補足する。
- Bさんの「アンケートをとった」行動を強調する。「自分の感覚だけじゃなく、他の人も困っているかを確認したこと」が問題を「機会」に変えた鍵だと伝える。
- 「AさんとBさんの違いを3つ」は板書で整理すると全体の学びが見える化される。

### グループワーク（最重要フェーズ）

**巡回のポイント**

- **PHASE 2「なぜの深掘り」** が最も重要で支援が必要なフェーズ。
  - 「なぜ」が1回で止まっているチームへ：「それはなぜ起きているんだろう？もう一回聞いてみよう」
  - 「便利じゃないから」という表面的な答えで止まっているチームへ：「便利でないと、誰がどんなとき困る？」と具体化させる。

- **PHASE 3「ペルソナ」** が「みんな」になりがちなチームへ：「1人に絞ろう。例えば、具体的に誰かの顔が思い浮かぶ？」と問いかける。ペルソナに名前・年齢・背景をつけさせると解像度が上がる。

- **アイデアが「アプリ・AI」だけになっているチームへ**：「テクノロジーを使わないとしたら？」「仕組みや人の行動で解決するとしたら？」と広げる。

- **マップが文字だらけになっているチームへ**：「矢印・丸・枠を使って！発表時に見やすい方がいい」と助言する。

- **一人がほぼ全部書いているチームへ**：「○○さんはどう思う？」と名指しで巻き込む。

### プレゼンテーション

- 「ペルソナ（具体的な人物）」が伝わるかどうかに注目して聴くよう全体に伝える。「あなたの周りにいる人が助かるサービス」というリアリティが価値を伝える鍵だから。
- 全体振り返りの問い2「3種類の価値でどのチームが多くカバーしている？」は軽く採点させると場が盛り上がる。
- フィードバックカード3色が発表チームに渡る瞬間を大切にする。「他のチームの目線が、あなたのアイデアをさらに育てる」という経験として強調する。

### 振り返りシート

- 問4「別の問題」への応用は「今日やったアイデアの"考え方"を使えそうな別の場面」と言い換えると書きやすい。
- 問3「重視する価値の種類」を全体でシェアすると「価値観の多様性」への気づきが生まれる。
- ★評価後、「全部★1の人？」→「全部★5の人？」と聞いて、両端の人に理由を聴くと場の学びが深まる。

## よくある問題と対処法

| 問題 | 対処 |
|------|------|
| 「不便なことが思い浮かばない」という生徒 | 「今朝の朝ごはん、もっとこうだったらいいのに、と思ったことある？」と生活の中を掘り下げる |
| 問題の掘り下げ（なぜ）が1回で止まる | 「それはなぜ起きているんだろう？もう一回聞いてみよう」を3回繰り返す |
| アイデアが「AIで解決」で止まる | 「AIが使えない場所・人のためのアイデアは？」と制約をつける |
| ペルソナが「みんな・すべての人」になる | 「1人だけ。名前をつけてみよう」と絞らせる |
| チームでアイデアがまとまらない | 「全部まとめなくていい。価値が一番伝わりやすいアイデアを1つ選ぼう」 |
| 時間が足りなくなる | PHASE 3を「ペルソナ名＋一言」に短縮。発表はチームの半数だけに |
| 振り返りシートに書けない生徒がいる | 「自分のマップを見ながら★だけ先につけて」と入口を下げる |

## SAILの5つの力との接続

| SAILの力 | このワークでの体験 |
|----------|------------------|
| **創造力** | 問題をアイデアに変え、価値を設計する。「ゼロからではなく問題から生まれる創造」を体験する |
| 自律力 | 自分で問題を見つけ、自分で価値を定義する。外からの評価ではなく「自己設計」が核心 |
| 挑戦力 | 「どうせ変わらない」を超えて、アイデアを出して表現する勇気 |
| 協働力 | チームで問題を共有し、互いのアイデアを高め合い、フィードバックで学ぶ |
| 表現力 | バリュー・デザイン・マップと発表・フィードバックカードで、アイデアを言語化・可視化する |
