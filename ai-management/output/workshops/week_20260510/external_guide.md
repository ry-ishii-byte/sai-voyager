**EntreComp領域**: 不確実性への対処

## ファシリテーションのポイント

### アイスブレイク

- 「振り返る前と後で見え方は変わった？」という問いを、全体への問いかけとして使う。これが本日のテーマの核心を体験させる導入になる。
- 「気づいたことがない」という参加者には「振り返ったとき、最初とは違う感想が出てきた？」と言い換える。
- ファシリ自身が「振り返りで見え方が変わった経験」を短く話すと、場が和らぐ。

### 導入講義

**スライド7（ループ図）の説明が核心。**

- 「ループ」を指でなぞりながら「この4ステップを意識してやっている人と、無意識に経験を流している人では、同じ経験をしても成長の差が出る」と伝える。
- 「コルブ（Kolb）」という名前は出さなくてもOK。「研究者が発見したサイクル」程度でよい。

**スライド8（3レベル）では自己アセスメントを促す。**

- 「自分は今、Lv.1〜3のどこにいると思う？」と手を挙げさせる。
- Lv.1が多くても「正直に言える！いい」と受け止める。今日のゴールはLv.3を体験することと伝える。

**スライド9（What/Why/So What）は本日のワークの骨格。**

- 「Whyの問いが一番難しい。でも一番大事」と強調する。
- 「なぜなぜ分析：最低3回Whyを繰り返す」を実演する。
  - 例：「授業で発表できなかった」→「なぜ？」→「準備が足りなかった」→「なぜ？」→「いつ準備するか決めていなかった」→「なぜ？」→「計画を立てるのが苦手だった」
  - 「こうして掘り下げると、"発表が下手"ではなく"計画が苦手"という本質の課題が見えてくる」

### ケーススタディ

- Aさんの「失敗→反省→同じ失敗」のループを板書で図示する（→矢印で循環を見せる）。
- BさんのLv.3ふり返りのプロセスを「①書き出す→②原因を特定する→③具体的な解決策を決める」と板書で分解する。
- 「Aさんが悪い人なのではなく、やり方を知らなかっただけ」と必ず補足する。

### グループワーク（最重要フェーズ）

**巡回のポイント**

- **PHASE 2「なぜ」の深掘り**が最も重要で、最も支援が必要なフェーズ。
  - 「なぜ」が1回で止まっているチームへ：「それはなぜ起きたと思う？もう一回"なぜ"を聞いてみよう」
  - 「努力不足」「真剣さが足りなかった」という表面的原因で止まっているチームへ：「じゃあ、なぜそのとき努力できなかったのかな？」とさらに掘り下げる。

- **PHASE 3「転移」**が空白になりがちなチームへ：「この発見、部活でも使えそうじゃない？」「家でも同じこと起きない？」と具体的な場面を提示する。

- **Mapが文字だらけ**になってきたら：「矢印や図も使って！プレゼン時に見やすい方が良い」と助言する。

- **一人がほぼ全部書いているチームへ**：「○○さんはどう思う？付箋に書いてみて」と他のメンバーを参加させる。

### プレゼンテーション

- 発表後にフィードバックカード（3色）を発表チームに渡す時間を30秒作る。この瞬間が「聴いた学びがアウトプットになる」体験になる。
- 全体振り返りのスライド19の問い3「今日一番ふり返りが深まった瞬間は？」は、必ず複数名から引き出す。場の盛り上がりのピークになる。

### 振り返りシート

- 問3「苦手なステップ」はワーク終了後に全体でシェアすると共感が生まれる。「②内省が苦手」「③抽象化が難しい」が多数派になりやすい。
- ★評価後、「3つ全部★5の人？」→手が上がれば「具体的にどこがうまくいった？」と引き出す。

## よくある問題と対処法

| 問題 | 対処 |
|------|------|
| 「ふり返る経験がない」という生徒 | 「今日の授業や、昨日の部活でもOK。最近の困ったことで十分」 |
| Journey Mapの「なぜ」が浅い（表面的） | 「それはなぜ？」をもう2回繰り返させる |
| プレゼンで内容が薄い | 「なぜそうなったか、もう少し教えてください」とフォロー質問を入れる |
| チームの意見がまとまらない | 「全部まとめなくていい。一番深い発見を1つ選ぼう」と絞らせる |
| 時間が足りなくなる | PHASE 3（転移）を「口頭で1つ言うだけ」に短縮する |
| 振り返りシートに書けない生徒がいる | 「Journey Mapを見ながら、★だけ先につけて」と入口を下げる |

## SAILの5つの力との接続

| SAILの力 | このワークでの体験 |
|----------|------------------|
| **自律力** | 自分の経験を自分でふり返り、学びを設計する。外からの評価ではなく「自己評価・自己成長」が核心 |
| 挑戦力 | 失敗をオープンにして、次につなげる勇気。「失敗 = 終わり」から「失敗 = データ」へ |
| 創造力 | Journey Mapで経験を俯瞰し、新たな視点・解釈を生む |
| 協働力 | チームで経験を共有し、互いの「なぜ」から学び合う |
| 表現力 | Journey Mapと発表・フィードバックカードで学びを言語化・可視化する |
